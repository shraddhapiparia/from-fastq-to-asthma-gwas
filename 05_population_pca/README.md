# 05: Population PCA

## Objective

Apply genotype quality control filters, compute principal components from LD-pruned SNPs, and visualize population structure. Produce eigenvectors used as covariates in the module 06 GWAS to control for ancestry-driven confounding.

---

## Why This Step Matters

Population stratification is one of the primary sources of false positives in GWAS. If cases and controls differ in ancestry — even subtly — variants that are common in one ancestry group will appear associated with disease simply because of that frequency difference, not because of any true biological effect. Principal component analysis (PCA) on genome-wide genotype data captures the major axes of ancestry variation across samples; including the top PCs as covariates in logistic regression absorbs this confounding signal.

PCA also serves as a sample-level quality check: duplicated samples, contaminated samples, and population outliers appear as anomalies in the PC space that would otherwise silently distort association statistics. Identifying and removing them before GWAS is a critical quality gate.

## Workflow Overview

```
Raw genotypes (datasets/)
    ↓
Genotype QC (PLINK)               — MAF, missingness, HWE filters; remove low-quality variants and samples
    ↓
LD pruning (PLINK)                — remove correlated SNPs to prevent over-representation of dense regions
    ↓
PCA (PLINK --pca 10)              — compute first 10 principal components from pruned SNPs
    ↓
Visualization (R)                 — PC1 vs PC2 scatter plot coloured by population; scree plot
    ↓
Eigenvectors                      → module 06 (GWAS covariates PC1–PC5)
```

### Running steps individually

#### Step 1 — Genotype QC

```bash
bash scripts/01_qc_plink.sh
```

Applies the following filters:

```bash
plink \
    --bfile ../datasets/data/demo_genotypes \
    --maf 0.01 \
    --geno 0.1 \
    --mind 0.1 \
    --hwe 1e-6 \
    --make-bed \
    --out results/demo_genotypes_qc
```

- `--maf 0.01`: removes variants with minor allele frequency < 1%
- `--geno 0.1`: removes variants missing in > 10% of samples
- `--mind 0.1`: removes samples missing > 10% of genotypes
- `--hwe 1e-6`: removes variants with Hardy-Weinberg equilibrium P < 1×10⁻⁶ (excess of heterozygotes suggests genotyping error)

#### Step 2 — LD pruning

```bash
bash scripts/02_ld_prune.sh
```

```bash
plink \
    --bfile results/demo_genotypes_qc \
    --indep-pairwise 50 5 0.2 \
    --out results/demo_genotypes_pruned

plink \
    --bfile results/demo_genotypes_qc \
    --extract results/demo_genotypes_pruned.prune.in \
    --make-bed \
    --out results/demo_genotypes_pruned
```

Scans in 50 kb windows, steps 5 kb, and removes one SNP from each pair with r² > 0.2. The `.prune.in` file lists the retained SNPs; `.prune.out` lists the removed ones.

#### Step 3 — PCA

```bash
bash scripts/03_run_pca.sh
```

```bash
plink \
    --bfile results/demo_genotypes_pruned \
    --pca 10 \
    --out results/demo_genotypes_pca
```

Computes the top 10 principal components using PLINK's randomized SVD implementation. Outputs `.eigenvec` (sample coordinates) and `.eigenval` (variance explained per PC).

#### Step 4 — Plot

```bash
Rscript scripts/04_plot_pca.R
```

Reads `.eigenvec` and population labels from the `.fam` file, merges them, and produces a PC1 vs PC2 scatter plot with per-population colouring and variance-explained axis labels, plus a scree plot of eigenvalues for all 10 PCs.

## How to Interpret Results

### PCA scatter plot (PC1 vs PC2)

- **Clusters** in the scatter plot correspond to populations with distinct allele frequency profiles. Well-separated clusters indicate detectable ancestry differences across samples.
- **Overlapping clusters** indicate low genetic differentiation between the simulated groups, expected with a small random dataset.
- **Sample outliers** (points far from the main cluster mass) indicate samples that differ substantially in ancestry from the rest of the cohort. These should be reviewed for possible contamination, sample mix-up, or genuine population difference.
- **Axis labels**: each PC axis shows the percentage of total genotype variance it explains. PC1 typically explains the most variance; declining values in the scree plot confirm that the first few PCs capture the dominant structure.

### Scree plot

- A sharp drop after PC1–PC3 indicates that a small number of components capture most of the ancestry structure — include those in GWAS.
- A flat scree (uniform eigenvalues) suggests no strong population structure, which is expected from random simulated data.

### QC log

- The PLINK `.log` file records how many variants and samples were removed at each filter. Unexpectedly large losses (>20% of variants) suggest data quality issues upstream.
- `qc_summary.txt` provides a concise tab-delimited record of post-QC counts for reproducibility checks.

---

## Decision Guidelines

| Observation | Action |
|-------------|--------|
| Samples clustering far from the main group | Remove as population outliers before GWAS, or restrict analysis to a homogeneous subset |
| Scree plot shows strong signal in PC1–PC3 | Include at least PC1–PC5 as GWAS covariates; more PCs if structure extends further |
| Scree plot is flat (no PC explains > 5%) | Include PC1–PC5 as a precaution; unlikely to see strong stratification inflation |
| HWE filter removes large fraction of variants | Check for batch effects or genotyping errors; may indicate platform-specific calling artefacts |
| Very few variants remain after LD pruning | Relax r² threshold (try 0.5) or check that input genotypes are not already a pre-pruned set |
| λ > 1.10 in module 06 QQ plot | Return here — add more PCs as covariates (PC6–PC10) or review whether outlier samples were removed |

**How many PCs to include in GWAS**: the standard approach is to include PC1–PC5 or PC1–PC10. For a 100-sample simulated dataset with modest structure, PC1–PC5 is sufficient. Use the scree plot to confirm that eigenvalues become negligible beyond the selected cutoff.

---

## What This Affects Downstream

- **Module 06 (GWAS)**: PC1–PC5 from `demo_genotypes_pca.eigenvec` are extracted and passed as covariates to PLINK logistic regression. Without these, any ancestry differences between simulated cases and controls inflate λ and produce false-positive associations.
- **Module 07 (PRS)**: the QC-filtered genotype fileset (`demo_genotypes_qc`) is the target cohort for PRS scoring. Samples removed here are absent from PRS output.
- **Cohort composition**: outlier removal decisions made here propagate to all subsequent genotype-based analyses. Any sample filtered out of the QC set is silently absent from GWAS, PRS, and eQTL annotation results.

---

## Key Concepts

**Why LD pruning before PCA**: regions of high LD (e.g., MHC, inversions, high-coverage repeat regions) would contribute disproportionately to the principal components if included. Pruning ensures that PC axes reflect genome-wide ancestry patterns rather than local LD structure.

**PC1 and PC2 as ancestry axes**: in datasets with multiple ancestral populations, PC1 typically separates the most divergent groups (e.g., African vs European in global studies). PC2 captures the next orthogonal axis of variation. In a homogeneous cohort, early PCs may reflect technical batch effects rather than ancestry.

**Population stratification and λ**: the genomic inflation factor λ (module 06 QQ plot) is the diagnostic for whether PCA adequately controlled stratification. A well-calibrated GWAS has λ close to 1.0. Values above 1.10 indicate residual confounding and warrant returning to this module to add more PCs or remove outlier samples.

**HWE filter**: Hardy-Weinberg equilibrium is expected at most loci in a random-mating population. Significant departure (P < 1×10⁻⁶) in controls is a flag for genotyping error, not disease association. HWE filters are applied to the QC step, not to GWAS, where case HWE deviation may reflect true association.

## Real-World Usage

In biobank-scale population studies:

- QC thresholds are stricter: MAF > 0.01–0.05, genotype missingness < 0.02, sample missingness < 0.02, HWE P < 1×10⁻⁶ in controls
- LD pruning uses tighter parameters (r² < 0.1) and excludes known high-LD regions (MHC, chromosome 8p inversion, chromosome 17q21) that dominate early PCs
- PCA is computed on a reference panel (1000 Genomes, gnomAD) and target samples are projected onto the reference PCA space for reproducible ancestry assignment
- Ancestry-matched cohort subsets (e.g., European-only, multi-ancestry) are then defined from the PC space before GWAS
- PC1–PC20 (or more) are included as covariates; mixed-model association methods (BOLT-LMM, SAIGE) handle residual relatedness that PCA covariates alone cannot absorb
